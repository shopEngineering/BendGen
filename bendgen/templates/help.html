BendGen Help

Bend Program Generator for Langmuir BendControl

BendGen is a desktop program generator for the Langmuir Systems BendControl press brake. It lets you create bend programs on your computer, export them as a ZIP file, and restore them onto your press brake via USB drive.

What BendGen does: Creates the same backup ZIP format that the BendControl uses. You build programs and bends in a web interface, export a ZIP, copy it to USB, and restore it on the machine — just as if you’d created the bends directly on the press brake touchscreen.

Quick Start Guide

Launch BendGen — double-click the BendGen executable. Your browser opens to http://localhost:5050.
Create a program — click + New Program in the left sidebar. Give it a name.
Add bends — click + Add Bend. Set the angle, material thickness, bend width, and select your die, punch, and material.
Export — click Export ZIP in the header. A backup ZIP file downloads.
Copy to USB — save the ZIP file to a USB drive.
Restore on machine — on the BendControl touchscreen, tap the backup/restore icon (⏳) in the upper-right toolbar, tap Restore From, and select the ZIP from USB. See Backup & Restore for details.

Programs & Bends

BendGen organizes work into programs, each containing one or more bends. This mirrors the BendControl structure exactly.

Creating a Program

  1. Click + New Program in the left sidebar.
  2. A new program appears in the list. Click it to select it.
  3. Type a name in the program name field at the top of the editor.
  4. Click Save Program to save.

Adding Bends

  1. With a program selected, click + Add Bend.
  2. Fill in the bend parameters (see Bend Fields below).
  3. Add as many bends as needed — each gets its own card or row.
  4. Use Duplicate to copy a bend with all its settings.
  5. Click Save Program when done.

Bend Fields Explained

Bend Angle (°)

The desired included angle of the bend — the angle between the two faces of the sheet after bending. A 90° bend creates a right angle. Smaller numbers = sharper bend.

Material Thickness

The thickness of the sheet metal being bent. Measure with calipers for best results. This value is critical — it directly affects bend depth calculations.

Bend Width

The length of the bend line across the workpiece — how wide the piece is where it contacts the die.

Punch Clearance

The gap between the punch tip and the top of the material before the bend stroke begins. This is the safety clearance plane — the punch descends at high speed until it reaches this distance above the material, then switches to slower bending speed.

Safety: Too small = not enough room to position the workpiece safely. Too large = wasted cycle time. A typical value is 0.1″.

Die / Punch / Material

Select the tooling and material for each bend. These must match what is physically installed on your machine. Add custom tooling via Manage Tooling.

Advanced Bend Settings

Click Advanced on a bend card to reveal additional fields.

Angle Compensation

Adjusts the bend angle to account for springback. If your material springs back 2°, set compensation to 2° with direction Underbent to overbend by that amount.

  • Underbent — actual bend is less than desired; machine bends further to compensate.
  • Overbent — actual bend is more than desired; machine bends less.

Additional Retract After Bend

Extra distance the ram retracts beyond the default position. Useful for clearance to flip or reposition the workpiece.

Back Gauge

Enable to set a precise back gauge position. The back gauge is a motorized finger behind the die that positions the workpiece at a consistent depth.

  • BG Finger — reference edge stop (G54–G59).
  • X Position — distance from die center to back gauge finger.
  • R Position — height of the back gauge finger.

Override Final Position

Manually set the exact ram depth, bypassing automatic calculation. Use with caution — incorrect values can damage tooling.

Notes

Free-text notes stored in the backup and visible on the BendControl touchscreen.

Managing Tooling

Click Manage Tooling in the header. BendGen comes with stock tooling preloaded, and you can add your own.

Dies

Each die has a name, height, and V-die opening.

Punches

Each punch has a name and height.

Materials

Each material has a name, tensile strength (PSI), inside radius rule of thumb, springback factor, and an optional gauge-to-thickness table. The gauge table auto-fills thickness when you select a gauge in the DXF importer.

Importing DXF Drawings

BendGen can analyze DXF flat-pattern drawings to automatically detect bend lines and create programs.

  1. Click Import DXF Drawing in the header.
  2. Select a .DXF file from your computer.
  3. The DXF preview modal opens showing the drawing with detected bend lines highlighted.
  4. Set your defaults (material, gauge/thickness, punch, die) in the right panel.
  5. Review and reorder bends in the Bend Order & Assignment table. Drag rows to reorder.
  6. Click Create Program to generate a program with all detected bends.

Tip: Hover a bend row in the planner to highlight it in the drawing. Click edges in the table to set the BG X reference per bend.

Importing Existing Backups

If you already have programs on your press brake, import them into BendGen to avoid overwriting them.

  1. On the BendControl, create a backup to USB (see Backup & Restore).
  2. In BendGen, click Import Existing Backup.
  3. Select the ZIP file from your USB drive.
  4. All programs and bends from the backup are loaded into BendGen.
  5. Add your new programs, then export as usual.

Important: Always import your machine’s current backup before adding new programs. Restoring a ZIP on the BendControl replaces all existing programs. Importing first ensures your export includes both old and new programs.

Exporting a ZIP

  1. Click Export ZIP in the header.
  2. A ZIP file downloads to your computer.
  3. Copy this ZIP to a USB drive.
  4. Restore it on the BendControl (see Backup & Restore).

Unit Switching (in / mm)

Use the in / mm toggle in the header to switch between inches and millimeters. All linear fields (thickness, width, clearance, positions) convert automatically. Angular values (degrees) are unaffected.

How it works: The BendControl stores all values in inches internally. The millimeter toggle is a convenience for users who prefer to work in metric — BendGen converts your mm entries to inches behind the scenes when saving. The exported ZIP always contains inch values, which is what the machine expects. You can switch between in and mm at any time without losing precision.

Form vs Table View

Toggle between Form and Table view using the buttons above the bend list.

  • Form view — each bend is a card with labeled fields. Best for editing one bend at a time.
  • Table view — all bends in a compact grid. Best for reviewing and comparing multiple bends.

Titan USB Bridge

The USB Drive Bridge lets you send bend programs directly from BendGen to the Titan press brake over WiFi — no USB drive swapping. A Raspberry Pi Zero W (or Zero 2W) plugs into the Titan’s USB port and acts as a virtual USB flash drive. BendGen talks to the Pi over your shop network.

How it works: The Pi presents itself to the Titan as a regular removable USB drive. When you click Deploy in BendGen, the Pi briefly “ejects” the drive, writes your file, and “reinserts” it. The Titan sees a freshly inserted USB drive with your program on it — ready to restore.

    │                 │    BendGen talks to Pi       │                  │
    │  BendGen app    │    on port 8080              │  USB Drive Bridge│
    │  (browser)      │                              │  (bridge.py)     │
    └─────────────────┘                              └──────┬───────────┘
                                                            │ USB cable
                                                            │ (data port)
                                                     ┌──────┴───────────┐
                                                     │   Titan Press    │
                                                     │   Brake          │
                                                     │                  │
                                                     │  Sees a USB      │
                                                     │  flash drive     │
                                                     └──────────────────┘

What You Need

  • Raspberry Pi Zero W or Raspberry Pi Zero 2W — these are the only Pi models with USB OTG on the data port. A Pi 4/5 will not work for this.
  • Micro SD card — 8 GB or larger. Class 10 or better.
  • Micro USB to USB-A cable — connects the Pi’s data port to the Titan’s USB port. Must be a data cable, not a charge-only cable.
  • Micro USB power supply (5V, 2.5A+) — powers the Pi via its power port.
  • Another computer — to flash the SD card and run BendGen.
  • Your shop WiFi network — both the Pi and your computer must be on the same network.

USB cable note: The Pi Zero has two micro USB ports. The one labeled “USB” (closest to the center) is the data port. The one labeled “PWR” (on the edge) is power only. You need to connect the data port to the Titan.

Step 1: Flash Raspberry Pi OS

Download Raspberry Pi Imager from raspberrypi.com/software. Install it on your computer (Windows, macOS, or Linux).
Insert the micro SD card into your computer (use an adapter if needed).
Open Raspberry Pi Imager and configure:
  • Device: Raspberry Pi Zero W (or Zero 2W)
  • Operating System: Raspberry Pi OS (32-bit) — the “Lite” version is fine (no desktop needed)
  • Storage: your SD card
Click the gear icon (or “Edit Settings”) to open advanced options. This is critical:
  • Set hostname: bendgen-bridge (so you can find it on the network as bendgen-bridge.local)
  • Enable SSH: check “Enable SSH” and select “Use password authentication”
  • Set username/password: pick something you’ll remember (e.g., user: pi, password: your choice)
  • Configure WiFi: enter your shop WiFi network name (SSID) and password. Set the country code.
Click “Write” and wait for the image to finish writing and verifying. This takes a few minutes.
Eject the SD card and insert it into the Pi Zero’s SD card slot (on the underside of the board).

Step 2: Install the USB Drive Bridge

Power on the Pi by connecting the power supply to the PWR micro USB port. Wait 1–2 minutes for the first boot (it will resize the filesystem and connect to WiFi).
SSH into the Pi from your computer. Open a terminal and run: 
ssh pi@bendgen-bridge.local

Accept the fingerprint prompt and enter the password you set in the Imager.

(If bendgen-bridge.local doesn’t resolve, see Find the Pi’s IP below.)

Download and run the installer: 
curl -sSL https://raw.githubusercontent.com/shopEngineering/BendGen_USB_drive_PiZero/master/install.sh -o install.sh
bash install.sh

The installer will:

  • Install Python and dependencies
  • Enable USB OTG gadget mode (dwc2 kernel overlay)
  • Create a 512 MB FAT32 disk image
  • Configure the USB mass storage gadget (configfs)
  • Install systemd services for automatic startup
Reboot when prompted. The installer will ask — say yes. After reboot, the Pi will automatically start the USB gadget and the bridge API.

After reboot: The bridge API starts automatically on port 8080. You can verify by SSH-ing back in and running: sudo systemctl status usb-bridge

Step 3: Connect to the Titan

Power off the Pi (unplug the power cable).
Connect the USB data cable from the Pi’s USB data port (center port) to any open USB-A port on the Titan.
Connect the power supply to the Pi’s PWR port (edge port).
Wait ~30 seconds for the Pi to boot. The Titan should detect a new USB drive. You can verify by checking if the Titan’s file browser shows a USB device.

Important: The Pi needs two cables — one for data (to Titan) and one for power. The Titan’s USB port may not supply enough power to run the Pi reliably. Always use a separate power supply.

Step 4: Find the Pi’s IP Address

You need the Pi’s network address to connect BendGen to it. There are several ways to find it:

Option A: Use the hostname (easiest)

If you set the hostname to bendgen-bridge during setup, try:

ping bendgen-bridge.local

If it responds, your bridge address is bendgen-bridge.local:8080

Option B: Check your router

Log into your WiFi router’s admin page (usually 192.168.1.1 or 192.168.0.1). Look for connected devices — find the one named bendgen-bridge and note its IP address.

Option C: SSH and check

If you can SSH into the Pi, run:

hostname -I

This prints the Pi’s IP address (e.g., 192.168.1.42). Your bridge address would be 192.168.1.42:8080.

Option D: Network scan

From your computer, scan the local network:

# macOS / Linux:
ping -c1 bendgen-bridge.local

# Windows:
ping bendgen-bridge.local

Static IP (recommended for permanent installs): To prevent the IP from changing, assign a static IP on your router (DHCP reservation) or configure it on the Pi itself. This way the bridge address never changes.

Step 5: Connect BendGen to the Bridge

Open BendGen on your computer (any computer on the same network as the Pi).
Click “Deploy” in the Titan section of the header. Since no bridge address is saved yet, a settings dialog opens.
Enter the bridge address — either bendgen-bridge.local:8080 or the IP address (e.g., 192.168.1.42:8080).
Click “Test Connection” to verify. You should see:
  • Connected — gadget active = everything is working
  • Connected — gadget not active = Pi is reachable but USB cable may not be connected, or gadget service didn’t start
  • Cannot reach bridge = wrong address, Pi is off, or not on same network
Click “Save”. The address is stored in your browser and persists across sessions.

Deploying & Importing via the Bridge

Deploying to Titan (BendGen → Titan)

  1. Create or edit your program in BendGen as usual.
  2. Click Deploy in the Titan section of the header.
  3. BendGen generates the backup ZIP, sends it to the Pi, and the Pi writes it to the virtual USB drive.
  4. The Titan sees a freshly inserted USB drive with your file.
  5. On the Titan touchscreen, tap Backup/Restore (⏳) → Restore From → select the ZIP.

Importing from Titan (Titan → BendGen)

  1. On the Titan touchscreen, create a backup to the USB drive (tap ⏳ → enter a name → Create Backup).
  2. In BendGen, click Get Backup in the Titan section of the header.
  3. A list of files on the USB drive appears. Click Import on the one you want.
  4. The backup is downloaded from the Pi and loaded into BendGen — all programs and bends appear in the editor.

Tip: The deploy cycle takes about 1–2 seconds. During that time the Titan briefly sees the USB drive disappear and reappear — this is normal. Wait for the “Deployed” confirmation in BendGen before touching the Titan.

Bridge Troubleshooting

BendGen says “Cannot reach bridge”

  • Make sure the Pi is powered on and connected to WiFi.
  • Verify the address — try pinging bendgen-bridge.local from your computer.
  • Make sure your computer and the Pi are on the same WiFi network.
  • Try using the IP address instead of the hostname (e.g., 192.168.1.42:8080).
  • SSH into the Pi and check the service: sudo systemctl status usb-bridge

Test says “gadget not active”

  • Check that the USB data cable is connected to the Pi’s center port (USB DATA), not the edge port (PWR).
  • Make sure the cable is a data cable, not a charge-only cable. Charge-only cables have no data wires.
  • Check the gadget service: sudo systemctl status usb-gadget
  • Check that dwc2 is loaded: lsmod | grep dwc2

Titan doesn’t see the USB drive

  • Try a different USB cable — some cables are power-only.
  • Try a different USB port on the Titan.
  • SSH into the Pi and check: cat /sys/kernel/config/usb_gadget/g1/UDC — it should show a device controller name.
  • Reboot the Pi with the USB cable connected.

Deploy succeeds but Titan doesn’t see the file

  • On the Titan, use Restore From (folder icon at bottom of backup popup) — not the regular backup list.
  • The file may be there but you need to browse to the root of the USB drive.

Pi loses WiFi or IP changes

  • Set a static IP reservation on your router for the Pi’s MAC address. This prevents the IP from changing.
  • Alternatively, always use the hostname bendgen-bridge.local instead of the IP.

Checking logs

SSH into the Pi and run:

# Bridge API logs:
journalctl -u usb-bridge -f

# Gadget setup logs:
journalctl -u usb-gadget

BendControl Overview

The BendControl is the Langmuir Systems touchscreen interface on the press brake. Key areas:

  • Top toolbar — Langmuir logo, connection status, version, and action buttons (firmware, bleed hydraulics, machine log, backup/restore, WiFi).
  • Left column — create bends, select tooling (punch/die/material), manage programs, run bend sequences.
  • Right column — live machine state: ram DRO, jog controls, LED indicators, homing, emergency stop.

Backup & Restore

This is the key workflow for getting BendGen programs onto your machine.

Finding the Backup Button

On the BendControl touchscreen, look at the top-right corner of the action bar. The backup/restore button has a clock/history icon (⏳). It sits after the machine log button and before the WiFi button.

Action bar layout (left → right): Version → Restart → System Info → spacer → App Update → Module FW → Machine FW → Bleed Hydraulics → Machine Log → ⏳ Backup/Restore → WiFi

Creating a Backup (machine → USB)

  1. Insert a USB drive into the BendControl.
  2. Tap the backup/restore icon (⏳) in the upper-right toolbar.
  3. A popup appears titled “Backups”.
  4. Type a name in the text field (e.g., “my-bends-march”).
  5. Tap Create Backup.
  6. The backup appears in the list. Take the USB to your computer.

Restoring a Backup (USB → machine)

Warning: Restoring replaces ALL programs on the machine. Always import your current machine backup into BendGen first so you don’t lose existing programs.

  1. Copy the BendGen-exported ZIP to a USB drive.
  2. Insert the USB into the BendControl.
  3. Tap the backup/restore icon (⏳) in the upper-right toolbar.
  4. At the bottom of the popup, tap Restore From (folder icon).
  5. A file browser opens. Navigate to and select your ZIP file.
  6. Note: The dialog may show “FIRMWARE” at the bottom — ignore this label. It is restoring bend programs, not firmware.
  7. After selecting the file, it appears at the top of the list as an “Auto...” entry. Tap Restore on that line.
  8. Your programs and bends are now loaded on the machine.

Backup Popup Layout

  • Header — “Backups” title with close (×) button.
  • Create section — text field + Create Backup button.
  • Backup list — scrollable list of existing backups (restore/delete each).
  • Footer — Restore From button (folder icon) for external files (USB).

Creating Bends on the Machine

The BendControl’s left column has a Create Bend view with these fields:

  • Desired Bend Angle (included angle in degrees)
  • Angle Compensation (with underbent/overbent toggle)
  • Punch-to-Material Clearance
  • Additional Retract After Bend
  • Material Thickness
  • Bend Width
  • Back Gauge (enable/disable, X/R position, jog speed)
  • Override Final Bend Position (enable/disable)
  • Punch / Die / Material selection

BendGen mirrors all of these fields, so programs created in BendGen are fully compatible with the machine.

Running a Program

On the BendControl, once a program is loaded:

  1. Select the program from the dropdown in the left column.
  2. The bend list shows all bends in order as cards.
  3. Tap a bend to select it as active.
  4. The machine displays calculated positions: Start Safety Clearance, Bend Clamp, Final Bend, Retract After Bend, Total Stroke, and Calculated Tonnage.
  5. Use the foot pedal and palm switches to execute the bend cycle (see the vertical position diagram).

Troubleshooting

BendGen won’t start

  • Make sure you extracted the ZIP first — don’t run the executable from inside the archive.
  • Windows: SmartScreen may warn “Windows protected your PC.” Click More info → Run anyway.
  • macOS: Right-click → Open the first time, then click Open in the dialog.

Browser doesn’t open automatically

Open your browser manually and go to http://localhost:5050.

Exported ZIP doesn’t restore on BendControl

  • Make sure the ZIP is on the root of the USB drive, not inside a folder.
  • Use the Restore From button (folder icon) at the bottom of the backup popup, not the regular backup list.

Programs missing after restore

Restoring a backup replaces all programs. Always Import Existing Backup in BendGen first to include your machine’s current programs in the export.

The “FIRMWARE” label appears during restore

Known cosmetic issue in the BendControl UI. Ignore it — it’s loading bend programs, not firmware.
